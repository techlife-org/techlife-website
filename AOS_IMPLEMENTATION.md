# AOS (Animate On Scroll) Implementation\n\nThis document describes how AOS (Animate On Scroll) has been integrated into the Techlife website template.\n\n## What is AOS?\n\nAOS (Animate On Scroll) is a library that allows you to animate elements as they scroll into view. It provides a simple way to add smooth, professional animations to your website without complex JavaScript.\n\n## Installation\n\nAOS has been installed in this project using npm:\n\n```bash\nnpm install aos\nnpm install @types/aos --save-dev\n```\n\n## Implementation\n\n### 1. CSS Integration\n\nThe AOS CSS has been added to `public/index.html`:\n\n```html\n<!-- AOS (Animate On Scroll) CSS -->\n<link href=\"https://unpkg.com/aos@2.3.1/dist/aos.css\" rel=\"stylesheet\">\n```\n\n### 2. JavaScript Initialization\n\nAOS is initialized in `src/App.tsx`:\n\n```javascript\nimport AOS from 'aos';\n\nReact.useEffect(() => {\n  // Initialize AOS\n  AOS.init({\n    duration: 1000, // Animation duration in milliseconds\n    easing: 'ease-in-out', // Easing function\n    once: true, // Whether animation should happen only once\n    mirror: false, // Whether elements should animate out while scrolling past them\n    offset: 100, // Offset (in px) from the original trigger point\n  });\n}, []);\n```\n\n### 3. Usage in Components\n\nAOS animations are applied using data attributes:\n\n```html\n<!-- Basic usage -->\n<div data-aos=\"fade-up\">\n  Content that will fade up when scrolled into view\n</div>\n\n<!-- With delay and custom duration -->\n<div \n  data-aos=\"zoom-in\" \n  data-aos-delay=\"300\" \n  data-aos-duration=\"1500\"\n>\n  Content with delayed zoom-in animation\n</div>\n```\n\n## Animations Added to Home Page\n\nThe following sections on the home page now include AOS animations:\n\n### Featured Services Section\n- **Animation**: `fade-up` with staggered delays\n- **Effect**: Services cards fade in from bottom with 100ms delay between each\n\n### About Section\n- **Left Column**: `fade-right` - Image section slides in from left\n- **Right Column**: `fade-left` - Text content slides in from right\n\n### Services Section\n- **Title**: `fade-up` - Section title fades in from bottom\n- **Service Cards**: `zoom-in` with staggered delays - Cards zoom in with 150ms delay between each\n\n### Portfolio Section\n- **Title**: `fade-up` - Section title fades in from bottom\n- **Project Cards**: `flip-left` with staggered delays - Cards flip in from left with 100ms delay between each\n- **Achievements**: `fade-up` with staggered delays - Achievement cards fade in with 200ms delay between each\n\n### Team Section\n- **Title**: `fade-up` - Section title fades in from bottom\n- **Team Cards**: `fade-up` with staggered delays - Team member cards fade in with 100ms delay between each\n- **Expertise Cards**: `slide-up` with staggered delays - Expertise cards slide up with 100ms delay between each\n\n### Contact Section\n- **Title**: `fade-up` - Section title fades in from bottom\n- **Call-to-Action**: `zoom-in` with delay - CTA section zooms in after title\n\n## Available Animation Types\n\n### Fade Animations\n- `fade-up` - Fade in from bottom\n- `fade-down` - Fade in from top\n- `fade-left` - Fade in from right\n- `fade-right` - Fade in from left\n- `fade-up-right` - Fade in diagonally\n- `fade-up-left` - Fade in diagonally\n- `fade-down-right` - Fade in diagonally\n- `fade-down-left` - Fade in diagonally\n\n### Flip Animations\n- `flip-left` - Flip from left\n- `flip-right` - Flip from right\n- `flip-up` - Flip from bottom\n- `flip-down` - Flip from top\n\n### Slide Animations\n- `slide-up` - Slide from bottom\n- `slide-down` - Slide from top\n- `slide-left` - Slide from right\n- `slide-right` - Slide from left\n\n### Zoom Animations\n- `zoom-in` - Zoom in from center\n- `zoom-in-up` - Zoom in from bottom\n- `zoom-in-down` - Zoom in from top\n- `zoom-in-left` - Zoom in from right\n- `zoom-in-right` - Zoom in from left\n- `zoom-out` - Zoom out from center\n- `zoom-out-up` - Zoom out to bottom\n- `zoom-out-down` - Zoom out to top\n- `zoom-out-left` - Zoom out to right\n- `zoom-out-right` - Zoom out to left\n\n## Configuration Options\n\n### Global Settings (in AOS.init())\n- `duration` - Animation duration (default: 1200ms)\n- `easing` - Easing function (default: 'ease')\n- `once` - Animate only once (default: false)\n- `mirror` - Animate out while scrolling past (default: false)\n- `offset` - Offset from trigger point (default: 120px)\n- `delay` - Global delay (default: 0ms)\n- `anchorPlacement` - Anchor placement (default: 'top-bottom')\n\n### Per-Element Attributes\n- `data-aos` - Animation type\n- `data-aos-delay` - Delay before animation starts\n- `data-aos-duration` - Animation duration\n- `data-aos-easing` - Easing function\n- `data-aos-once` - Animate only once\n- `data-aos-offset` - Offset from trigger point\n- `data-aos-anchor` - Use different element as trigger\n- `data-aos-anchor-placement` - Anchor placement\n\n## Demo Page\n\nA comprehensive demo page has been created at `/aos-demo` that showcases:\n- All available animation types\n- Different duration and delay examples\n- Various easing functions\n- Implementation code examples\n- Configuration options\n\n## Best Practices\n\n1. **Performance**: Use `once: true` for animations that don't need to repeat\n2. **Accessibility**: Consider users who prefer reduced motion\n3. **Timing**: Use appropriate delays to create smooth, sequential animations\n4. **Subtlety**: Don't overuse animations - they should enhance, not distract\n5. **Testing**: Test on different devices and screen sizes\n\n## Browser Support\n\nAOS works in all modern browsers that support:\n- CSS3 transforms\n- CSS3 transitions\n- IntersectionObserver API (with polyfill for older browsers)\n\n## Troubleshooting\n\n### Animations not working?\n1. Check that AOS CSS is loaded\n2. Ensure AOS.init() is called\n3. Verify data-aos attributes are correct\n4. Check browser console for errors\n\n### Performance issues?\n1. Reduce the number of animated elements\n2. Use `once: true` for one-time animations\n3. Increase the offset to trigger animations earlier\n4. Consider using `will-change: transform` CSS property\n\n## Resources\n\n- [AOS GitHub Repository](https://github.com/michalsnik/aos)\n- [AOS Documentation](https://michalsnik.github.io/aos/)\n- [CSS Easing Functions](https://easings.net/)\n"
